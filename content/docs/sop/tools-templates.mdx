Standardized tools and templates ensure consistency across all customer projects.

## Required Tools

| Tool | Purpose |
| :---- | :---- |
| **Asana** | Project management, task tracking, documentation, and templates |
| **Figma** | All design work and collaboration |
| **Loom** | Daily updates and feedback |
| **Slack** | Quick questions only (not for detailed feedback) |
| **Fireflies** | Meeting recordings |
| **Cal.com** | Scheduling (all team members must have an active account) |

### Slack Channel Naming Standard

All internal Slack channels must use the `team-xxx` naming convention (e.g., `team-operations`). This ensures:
- Global consistency
- Easy searchability
- Predictable structure across all departments

## Asana Implementation Requirements

- All customer projects must have standardized Asana templates
- Daily task tracking and progress updates (with deadlines included)
- Meeting notes and decisions documented in Figma
- Associate questions from meeting recordings tracked in Slack
- Shared project boards between Lead and Associate

## Where to Find Templates

Templates are maintained by Alex (DesignOps Lead):

**Weekly maintenance:**
- Check: Are teams using templates?
- Check: Are Associates submitting EOD updates?
- Flag issues to Leads/HOD early

**Monthly maintenance:**
- Run retrospective (Start/Stop/Continue)
- Review SOP improvements
- Update templates based on feedback
- Report model health to HOD

**Quarterly maintenance:**
- Survey all Leads + Associates
- Analyze metrics (satisfaction, capacity, growth)
- Propose major SOP changes
