# Contributing

## Branch Naming Convention

```
<username>/<type>-<short-description>
```

### Types

| Type       | Use for                                    |
| ---------- | ------------------------------------------ |
| `feat`     | New features                               |
| `fix`      | Bug fixes                                  |
| `refactor` | Code refactoring (no new features or fixes)|
| `docs`     | Documentation changes                      |
| `chore`    | Maintenance tasks, dependencies, configs   |
| `test`     | Adding or updating tests                   |
| `style`    | Formatting, whitespace, no code changes    |

### Examples

```
aalter/feat-user-authentication
aalter/fix-login-redirect
aalter/refactor-api-client
aalter/docs-readme-update
aalter/chore-update-dependencies
```

### Rules

- Use lowercase
- Use hyphens to separate words (not underscores)
- Keep descriptions short but descriptive (2-4 words)
- No special characters except `/` and `-`
